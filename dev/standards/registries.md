# Registry Standard

Conventions for first-class resource registries in this project. One page. Grounded in the shape of the registries we have, not speculation.

## When to create one

Open a registry when you have a designer-authored `Resource` type backed by `.tres` files under `data/tres/<type>/`. Skip registries for:

- Runtime-only types (no `.tres` backing).
- Resources embedded inside other resources.
- Small closed sets that belong in constants or enums.

If a type meets the bar, the registry is the single source of truth for "does this X exist" and "what are all the Xs". Don't reconstruct that answer by scanning a different registry.

## File layout

| Thing                 | Location                                                              |
| --------------------- | --------------------------------------------------------------------- |
| Script                | `global/autoloads/registries/<singular>_registry.gd`                  |
| Resource files        | `data/tres/<plural>/`                                                 |
| Path constant         | `DataPaths.<PLURAL>_DIR`                                              |
| Autoload registration | `project.godot`                                                       |
| Lifecycle             | load resources in `_ready()` via base class                           |

If the `.tres` directory is generated by tooling and gitignored, say so in a comment next to the `DataPaths` constant — a fresh checkout without the generator run will hit an empty registry and fail at boot.

## Base class

All registries extend `ResourceRegistry` (`global/autoloads/registries/resource_registry.gd`). Override two methods:

```gdscript
extends ResourceRegistry

func _dir_path() -> String:
    return DataPaths.<PLURAL>_DIR

func _id_of(r: Resource) -> String:
    return (r as <X>Data).<x>_id if r is <X>Data else ""
```

The base class handles `_ready()` loading and default `validate()` (non-empty check). Override `validate()` to add cross-reference checks.

`validate()` checks **authored content only** — that designer `.tres` files are well-formed and reference ids that exist. It **must not read live/save state**; validating restored save state (live ids still resolve) is the save provider's job, in its `from_dict` / `validate` (see the Save System section of `CLAUDE.md`).

## Required API

Every registry exposes, for its resource type `T`:

- `get_<singular>_by_id(id: String) -> T` — O(1) lookup. Returns null on miss.
- `get_all_<plural>() -> Array[T]` — typed wrapper over `get_all()`. Primary iteration API.
- `size() -> int` — inherited from base class; used by `validate()`.

`get_all()` on the base class returns untyped `Array`. The typed `get_all_<plural>()` wrapper on the concrete class is what callers should use.

**Iterate resources, not ids.** The `_ids` form exists for save/load boundaries, not as the default iteration path.

**Pass Resource refs at non-serialization call sites.** String ids belong at serialization boundaries; refs belong in game logic.

## Forbidden APIs

- **No display-name wrappers.** `get_<singular>_display_name(id)` is banned. Callers take the resource and read `.display_name`.
- **No fallback-to-id accessors.** Any helper of the form "return field X, or the id if missing" hides data bugs.
- **No cross-registry scanning.** Scanning Registry A to answer Registry B's question is the anti-pattern this standard exists to prevent.

## Inverse lookups

When registry A needs to answer "which Bs belong to this A?", prefer in order:

1. **Direct reference on the resource.** If `CategoryData` holds `super_category: SuperCategoryData`, answer via `cat.super_category`.
2. **Index built in `_ready`.** If grouping is needed, build a `Dictionary` at startup after the dependency registry has loaded.
3. **Never by scanning a third registry.**

If your `_ready` builds an index by walking another registry, add `assert(OtherRegistry.size() > 0, "...")` at the top and verify autoload order in `project.godot`.

## Cross-registry method naming

When one registry exposes a lookup keyed by **another** registry's id, use the `_for` suffix:

```
CategoryRegistry.get_super_category_for(category_id) -> SuperCategoryData
```

Rule: if a method's parameter id-type differs from the registry's own id-type, the name ends in `_for`.

## Checklist for a new registry

1. Script at `global/autoloads/registries/<singular>_registry.gd`, extending `ResourceRegistry`.
2. `DataPaths.<PLURAL>_DIR` constant. Comment if gitignored.
3. Autoload line in `project.godot`, before `SaveManager`. Check load order if `_ready` reads other registries.
4. Override `_dir_path()` and `_id_of()`. Override `validate()` for cross-reference checks.
5. Required API only: `get_<singular>_by_id`, `get_all_<plural>`, `size`. No display-name wrappers, no fallback accessors.
